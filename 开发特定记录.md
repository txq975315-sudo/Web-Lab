## 一、项目概述

### 1.1 项目定位
Kairos Thinking Lab 是一个"刻意练习"场域的个人思考工具，通过 AI 辅助帮助用户进行深度商业思考和决策。

### 1.2 核心设计理念
- **从"记录型"到"实验室型"**：让 AI 的每一次洞察都转化为可执行的认知资产
- **刻意练习**：用户自己在练习，而非教练替你打比赛
- **决策可追溯**：每一个决定都有依据、时间戳和替代方案

### 3.3 专家席模式切换 ✅

**功能描述**：支持在"压力测试模式"和"深度引导模式"之间切换，改变 AI 的对话风格。

**实现位置**：`src/context/LabContext.jsx`、`src/components/LabPanel.jsx`

**两种模式对比**：

| 维度 | 压力测试模式 | 深度引导模式 |
|------|-------------|-------------|
| AI 角色 | 外部审查者 | 苏格拉底式教练 |
| 输出形式 | 直接指出漏洞 + 致命追问 | 层层追问引导用户自己思考 |
| 用户角色 | 接收者 | 思考者 |
| System Prompt | 挑战者视角，攻击假设 | 引导者视角，用问题代替答案 |

**深度引导模式规则**：
1. 永远不要直接说出结论——用问题引导对方自己抵达
2. 当对方给出一个主张，追问其前提假设
3. 当对方回避一个方向，温和地重新打开那个方向
4. 当对方形成洞察，要求其清晰表述并记录
5. 每轮结束询问"你现在怎么理解这个问题？"

**核心函数**：
```javascript
// 1. switchExpertMode - 切换专家模式
switchExpertMode(mode) {
  setExpertMode(mode) // 'pressure' | 'guided'
}

// 2. getExpertModePrompt - 获取当前模式的 System Prompt
getExpertModePrompt() {
  if (expertMode === 'guided') {
    return 深度引导模式 Prompt
  }
  return 压力测试模式 Prompt
}
```

**Context 状态**：
```javascript
const [expertMode, setExpertMode] = useLocalStorage('kairos-expert-mode', 'pressure')
```

### 3.5 召唤导师按钮（上下文注入器）✅

**功能描述**：在任何文档编辑页面，点击"✨ 召唤导师"按钮，AI 自动针对当前模块进行深度追问。

**实现位置**：`src/config/aiPrompts.js`、`src/context/LabContext.jsx`、`src/components/ArchivePanel.jsx`、`src/components/LabPanel.jsx`

**核心函数**：
```javascript
// 1. summonMentor - 召唤导师
summonMentor(docId) {
  // 1. 获取当前文档内容
  // 2. 获取项目宪法作为背景约束
  // 3. 构建包含上下文的消息
  // 4. 设置 labMessageToSend 和 autoSendLabMessage
  // 5. 切换到深度引导模式
}

// 2. LabPanel 中处理预填充消息
useEffect(() => {
  if (labMessageToSend) {
    setInputValue(labMessageToSend)
    if (autoSendLabMessage) {
      // 自动点击发送按钮
    }
  }
}, [labMessageToSend, autoSendLabMessage])
```

**数据流转**：
```
用户点击"✨ 召唤导师"按钮
    ↓
summonMentor(docId) 被调用
    ↓
收集文档内容 + 项目宪法背景
    ↓
buildSummonMentorMessage() 构建消息
    ↓
setLabMessageToSend() + setAutoSendLabMessage(true)
    ↓
LabPanel useEffect 检测到预填充消息
    ↓
自动填入输入框 + 自动发送
    ↓
AI 以深度引导模式开始追问
```

**UI 交互**：
1. **ArchivePanel**：文档操作按钮区域显示"✨ 召唤导师"按钮
2. **LabPanel**：输入框自动填充召唤导师消息
3. **自动发送**：消息自动提交给 AI，AI 立即开始深度追问
4. **模式切换**：自动切换到"深度引导模式"

**Context 新增状态**：
```javascript

const [autoSendLabMessage, setAutoSendLabMessage] = useState(false)
```

## 九、开发规范

### 9.1 代码风格
- 使用 React Hooks
- 使用 Tailwind CSS 进行样式设计
- 使用 Framer Motion 实现动画
- 状态管理使用 Context API + useLocalStorage

### 9.2 组件拆分原则
- 每个组件只负责单一功能
- 组件之间的状态共享通过 Context
- 复杂的业务逻辑封装在 Context 或独立工具函数中

### 9.3 状态管理规范
- 持久化状态使用 `useLocalStorage`
- 临时状态使用 `useState`
- 派生状态使用 `useMemo` 或 `useCallback`

### 9.4 文件命名规范
- 组件文件：PascalCase（如 `LabPanel.jsx`）
- 工具函数：camelCase（如 `linkParser.js`）
- 配置文件：camelCase（如 `templates.js`）
