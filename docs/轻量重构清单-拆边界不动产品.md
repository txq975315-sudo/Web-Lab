# 轻量重构清单：只拆边界、不动产品

> **目标**：降低耦合、缩小单文件、为新功能（压力测试等）预留挂载点；**每一步合并后行为与 UI 须与重构前一致**。  
> **方法**：纯移动/重导出/包装器；不改业务分支、不改文案、不改默认数据；每步跑 `npm test` + 手动点一遍核心路径。

---

## 执行原则（必读）

1. **一步一小 PR**：宁可 5 个 50 行 PR，不要 1 个 500 行 PR。
2. **先加后减**：新路径文件先落地并让旧路径 `import` 新路径；确认无引用旧路径后再删重复（若保留 re-export 则不必删）。
3. **禁止夹带**：重构 PR 里不写新功能、不修 bug、不调样式（除非移动文件导致路径变化所必需的相对路径修正）。
4. **验收**：`npm test` 全绿；本地 `npm run dev` 验证：Landing → Dashboard → Live 演练发一条消息；成长教练进一页；考古列表能开；工作台切换工具栏无报错。

---

**目的**：Context 文件过长；将**不依赖 React、不闭包捕获 dispatch** 的树遍历/纯工具移到 `utils/`，减少阅读噪音。

|---|------|------|
| C.1 | 盘点纯函数 | 如 `findNodeById`、`collectDocuments`、`updateNodeInTree`、`addNodeToParent`、`removeNodeFromTree` 等（已在文件顶部附近）。 |

| C.4 | 单测（可选但推荐） | 对 `projectTreeOps` 写 2～3 个最小用例（增删改查节点），防止以后误改。 |
| C.5 | 验证 | 创建项目/文档、树展开、删除节点、切换文档与重构前一致。 |

**完成标准**：`LabContext.jsx` 缩短；业务路径无变化；`npm test` 通过。
