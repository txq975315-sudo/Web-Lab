# 压力练习工作台迭代 — 开发记录与对接备忘

> 文档用途：与 PRD、后续功能（打字机、200 字限制等）及云端同步方案对齐时，快速了解本轮已落地范围与代码触点。  
> 对应提交：`050c2c6`（`feat(压力练习): 工作台统一结构化引擎、历史分区、项目解绑与专注布局`），以远端 `main` 为准。

---

## 一、需求与结论摘要

| 主题 | 说明 |
|------|------|
| **统一引擎** | 压力练习（练）工作台 `live` + `workbenchLayout` 下，移除内嵌 **LiveLab** 旧追问分支；仅保留 **PressureTestWorkbench** → **PressureSessionRunner**，数据与逻辑走 **pressureSessionStore** / **pressureApi**（拆解、三轮九问、盲区报告）。 |
| **历史入口** | 已保存会话迁入「**历史压力练习**」：**PressureSessionHistoryPanel** 按**日历日**分组、可展开收起；工作台**中间栏**选中该工具时与 **WorkbenchLeftPanel**（非 live 左浮层）共用同一组件；列表随 **localStorage** 写入刷新（自定义事件 `thinking-lab-pressure-sessions-changed` + `storage`）。删除 **PressureSessionList.jsx**。 |
| **项目绑定** | **LabContext.switchLabMode**：进入 `live` 时暂存当前 `activeProjectId` 并清空项目；离开 `live` 到非考古模式且台内未手动选项目时恢复暂存；从 `live` 进入考古时把「当前项目或暂存项目」写入考古用 `previousProjectId`。非工作台顶栏「实时演练」仍可能有独立 LiveLab 与项目逻辑，未在本轮一并收束。 |
| **跨栏续做** | **continuePressureSession**、**consumePressureResumeSessionId**、**pressureWorkbenchActiveSessionId**（由 **LabPanel** 在 live 工作台同步 Runner 会话 id）：历史列表「继续」可切到压力练并打开会话；中间栏高亮当前会话。 |
| **专注布局** | **非**整页实色全屏 Portal：会话进行中 **App** 在存在 `pressureWorkbenchActiveSessionId` 时为 `wb-workbench-row` 增加 **`wb-pressure-session-active`**，并隐藏 **RightInsightDeck**；**CSS** 隐藏中间项目/历史列、内联文档与分隔条，主列拉满；**wb-pressure-dialog** 仍使用原有磨砂与 **`var(--wb-thread-max)`** 居中宽度，**保留 wb-shell 云底与透明度**。 |
| **Runner 交互** | **PressureSessionRunner** 内联于对话框；**Esc** 与 **「结束练习」** 走 **requestExit**；未完成会话退出前 **`confirm`**；曾实现 `document.body` + Portal 全屏，按产品反馈撤回。 |
