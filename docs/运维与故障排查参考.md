## 遇到问题时的查找顺序

### 1. 线上打不开 / 页面还是旧的

- 使用浏览器 **无痕模式** 或 **强制刷新**（Ctrl+F5）。
- 打开仓库 **Actions**，确认最近一次 **Deploy GitHub Pages** 为 **绿色成功**；若仍在运行或失败，等成功或先看失败日志。
- 若你改过 GitHub **仓库名**，需同步修改 **`.github/workflows/deploy-pages.yml`** 里的 **`VITE_BASE:/仓库名/`**，否则静态资源路径错误会导致白屏或 404。

### 2. GitHub Actions 失败（红色）

- 点开失败的那次运行 → 展开 **红色** 的步骤 → 阅读报错最后一屏英文。
- **常见原因：**
  - **`npm ci` 失败**：仓库根目录缺少 **`package-lock.json`**，或未提交到 Git。
  - **部署被拒绝**：仓库 **Settings → Environments → github-pages** 的分支规则不允许当前分支；可将 **Deployment branches** 设为 **No restriction**（个人项目），或为 **`main`** 增加允许规则。
  - **权限**：**Settings → Actions → General → Workflow permissions** 建议为 **Read and write**。

### 3. 本地开发连不上 http://localhost:5173

- 必须在项目根目录先执行 **`npm run dev`**，且 **保持该终端窗口不关**；关掉即停止开发服务器。
- 若端口被占用，以终端里 Vite 打印的 **Local** 地址为准（可能不是 5173）。

## 数据与备份（简要）

- 工作台数据在 **浏览器 localStorage**，换电脑/浏览器 **不会自动同步**。
- 应用内 **设置** 已提供 **导出工作台备份 / 导入备份**（整站 localStorage 快照），重要内容请定期导出保存。
