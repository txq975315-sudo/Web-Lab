## 一、模块概述与核心决策

### 1.1 产品定位

成长教练是 Thinking Lab「练 · 学 · 看」闭环中的「学」模块。它承接压力测试暴露的思维盲区，通过**每日方法论推送**、**知识卡片**、**练习题**、**AI评分**、**模拟追问**、**能力雷达图**六个环节，帮助用户从「知道问题」到「真正掌握方法论」，最终形成可迁移的结构化思考能力。

### 1.2 核心决策（基于本次讨论的最终结论）

| 决策项 | 结论 | 原因 |
|--------|------|------|
| **方法论数量** | **7个**（新增「约束权衡法」） | 覆盖产品全生命周期，含技术可行性评估 |
| **模板数量** | **29种**（24个现有 + 5个新增） | 补齐变现/留存/运营/技术可行性盲区 |
| **训练模式** | **两种并存**：项目绑定 + 独立训练 | 项目绑定=构建，独立训练=提升 |
| **独立训练案例** | **必须是真实项目/产品** | 虚拟案例用户无法验证 |
| **知识卡片定位** | **独立「每日方法论」入口** | 每日推送1张，培养打开习惯 |
| **模板-方法论映射** | **「1主+1辅」双方法论绑定** | 竞品分析不只是拆解，还要预判对手 |
| **V2用户画像** | **预留3-5年经验PM扩展位** | V1验证后，通过新增模板和方法论深度扩展 |

### 1.3 两种训练模式定义

| 模式 | 场景 | 案例A（主体） | 案例B（参照） | 产出 | 保存位置 |
|------|------|-------------|-------------|------|---------|
| **项目绑定训练** | 我有一个想法，想系统性想清楚 | 用户自己的项目/想法 | 用户选的真实竞品 | 完整项目文档 | 项目树 `/项目名/模块/文档` |
| **独立训练** | 我想单独提升某个能力 | 用户选的真实产品（自己常用或行业标杆） | 用户选的分析对象 | 完整分析报告 | `exerciseHistory` + 可导入项目树 |

**两种模式共享**：方法论维度分数、4档提示强度体系（按累计得分）、AI评分与反馈逻辑。

**关键设计**：独立训练过程中随时可点击「绑定到项目」。

### 1.4 模块架构图

```
成长教练（大模块）
  │
  ├── 📚 每日方法论（独立内容入口）
  │     ├── 今日推送（1张卡片，3分钟阅读）
  │     ├── 方法论归档（7个方法论 × N个案例）
  │     ├── 我的收藏
  │     └── 行业热点（今日案例 + 方法论拆解）
  │
  ├── 🏗️ 项目绑定训练
  │     └── 训练地图 → 选择模板 → 知识卡片 → 练习 → 评分 → 保存文档
  │
  └── ⚡ 独立训练
        └── 选择模板/真实案例 → 知识卡片 → 练习 → 评分 → 保存练习记录
```

## 二、七大方法论体系

### 2.1 设计哲学

29个文档模板归属于7套底层思考结构。每个模板绑定**1个主方法论**（主导思维方式）+ **1个辅方法论**（补充视角）。

**方法论名称规则**：每个方法论都有名字（方便调用知识结构），但每个名字的出场方式因方法论而异——有的先抛、有的后点、有的藏在工具里。

### 2.2 方法论总览

| # | 方法论 | 一句话 | 记忆钩子 | 呈现方式 | 名称位置 |
|---|--------|--------|---------|---------|---------|
| 1 | **多维拆解法** | 把复杂对象拆成维度逐个分析 | 同时想多个方面 | 场景类比 | 开头 |
| 2 | **场景代入法** | 进入用户场景感受friction找痛点 | 进入那个场景 | 追问引导 | 中间 |
| 3 | **第一性推理** | 追问为什么追到根本原因 | 追到根上 | 对比反差 | 结尾 |
| 4 | **推演博弈法** | 假设对手反应推演多步选最优 | 填完3轮就知道 | 工具化（决策表） | 给工具时 |
| 5 | **闭环验证法** | 假设→最小验证→数据→迭代 | 建一点测一点 | 故事化 | 高潮处 |
| 6 | **指标拆解法** | 拆核心指标找杠杆 | 拆开找杠杆 | 工具化（拆解树） | 解决后 |
| 7 | **约束权衡法**（新增） | 资源有限先做哪件 | 先做ROI最高的 | 工具化（ICE矩阵） | 给工具时 |

### 2.3 新增方法论详解：约束权衡法

**为什么需要？**

现有6个方法论没有覆盖「资源约束下的取舍决策」——这是MVP范围界定、技术可行性评估、资源分配的核心能力。

**呈现方式**：

> 你只有100万预算、3个月时间、5个人。想法有10个，先做哪个？
>
> 填这个ICE表：
> ```
> ┌──────────┬──────────┬──────────┬──────────┐
> │  想法     │ Impact   │ Confidence│ Ease    │
> │           │ 影响力   │ 把握度    │ 容易度  │
> ├──────────┼──────────┼──────────┼──────────┤
> │ 功能A    │    8     │    9      │    7    │
> │ 功能B    │    5     │    3      │    4    │
> └──────────┴──────────┴──────────┴──────────┘
> ```
> 总分 = I×C×E，先做分数最高的。这个工具叫【约束权衡法】。

### 2.4 方法论呈现方式详解（避免千篇一律）

**多维拆解法 — 场景类比（开头给名字）**：

> 先说一个你本来就会的本能：挑手机时，你会同时想「贵不贵？好不好买？拍照好不好？用久了卡不卡？」这个「同时想多个方面」的本能就是【多维拆解】。$APPEALS只是把它变成了8个维度。

**第一性推理 — 对比反差（结尾点破）**：

> 先看一个反例：「这个App需要加社交功能」（这是结论还是推理？）再看一个正例：「用户说孤独 → 孤独=需要连接？→ 连接=社交功能？→ 连接=被看见？→ 也许『展示』比『聊天』更重要」。区别在哪？前者停在表面，后者不停追问「为什么」。这种「追到根上」的思考方式，我们叫它【第一性推理】。

**推演博弈法 — 工具化（直接给决策表）**：

> 做任何决策前，先填这个3×3表。填完3轮，最佳策略通常就明显了。这个工具叫做【推演博弈法】。

**约束权衡法 — 工具化（ICE矩阵）**：

> 资源有限时，不做「想做的」，做「该做的」。用ICE评分：Impact × Confidence × Ease。这个工具叫【约束权衡法】。

## 四、知识卡片模块（含每日方法论）

### 4.1 定位：独立的「每日思维补给」入口

知识卡片**不是练习的附属品**，而是独立的每日内容产品。即使用户不练习，也能每天获得思维训练。

### 4.2 两种形态

| 形态 | 场景 | 内容 | 时长 |
|------|------|------|------|
| **每日推送** | 用户打开成长教练 | 今日行业热点 + 方法论拆解 | 3分钟 |
| **知识卡片** | 练习前预习 / 主动查阅 | 完整四层结构（概念+案例+方法论+自检） | 5-10分钟 |

### 4.3 每日推送界面

```
📅 2025-06-15 每日方法论 · 第47期

🔥 今日行业热点：小红书切断外链，力推站内闭环电商

💡 用「第一性推理」拆解：
   为什么小红书不做导流，要做闭环？
   → 追问1：导流的本质是什么？（流量变现，一次性）
   → 追问2：闭环的本质是什么？（用户资产沉淀，持续性）
   → 追问3：小红书的终局是什么？（内容+交易的超级生态）

💭 今日自检：如果你是拼多多PM，你会怎么应对？

[深入了解第一性推理 →]  [用这个方法练一次 →]
```

### 4.4 知识卡片四层结构（练习前）

| 层次 | 内容 | 说明 |
|------|------|------|
| **Level 1** | 一句话概念 | 用大白话解释，不要术语 |
| **Level 2** | 一个具体案例 | 锚定应用场景，真实商业案例 |
| **Level 3** | 方法论总结 | 差异化呈现（因方法论而异） |
| **Level 4** | 自检清单 | 3-5个检查点 |

### 4.5 内容生成策略

- **热门知识点**：预配置（`knowledgeTopics.js`），加载零延迟
- **每日推送**：AI实时生成（结合当日行业热点）
- **长尾知识点**：AI实时生成，缓存结果

### 4.6 数据流

```
每日推送：
  系统定时生成（或用户打开时生成）
    → 抓取今日行业热点（或从预设热点库选取）
    → AI生成「热点+方法论拆解」
    → 展示给用户
    → [深入了解] → 进入完整知识卡片
    → [练一次] → 进入相关模板练习

知识卡片（练习前）：
  用户选择模板
    → 系统查询预配置库
    ├── 命中 → 直接展示
    └── 未命中 → AI生成 → 缓存 → 展示
    → [开始练习] → 进入练习题
```

```

## 五、两种训练模式

### 5.1 模式一：项目绑定训练

**定义**：用户有一个想法/项目，想系统性地把它想清楚。练习结果自动构建项目文档库。

**案例真实性**：案例A=用户自己的项目，案例B=用户选的真实竞品。

**训练地图**：

```
📁 专注App（由成长教练创建）
  ├── 📄 核心定位（manifesto）— 自动创建
  ├── 📁 01 市场与用户洞察
  │     ├── 📄 竞品分析 [3档·基本掌握]（累计7.3分）
  │     ├── 📄 用户画像 [1档·需要引导]（累计2.1分）
  │     └── 📄 价值主张画布 🔒（需完成上面两项解锁）
  ├── 📁 02 策略与增长
  │     ├── 📄 商业画布 [2档·渐入佳境]（累计5.4分）
  │     └── 📄 北极星指标 🔒（需商业画布✅解锁）
  ├── 📁 03 变现与商业化（V1新增模块）
  │     ├── 📄 定价策略 🔒（需竞品分析✅+商业画布✅解锁）
  │     └── 📄 变现模型 🔒（需定价策略✅解锁）
  ├── 📁 04 执行路线图
  │     └── 📄 MVP范围 🔒（需商业画布✅解锁）
  ├── 📁 05 增长与运营
  │     ├── 📄 留存分析 🔒（需商业画布✅+北极星指标✅解锁）
  │     └── 📄 运营SOP 🔒（需留存分析✅解锁）
  └── 📊 能力雷达图（7维度进度可视化）
```

**解锁规则**（部分示例）：

| 模板 | 前置条件 |
|------|---------|
| 竞品分析 | 无 |
| 用户画像 | 无 |
| 价值主张画布 | 竞品分析✅ + 用户画像✅ |
| 商业画布 | 价值主张画布✅ |
| 定价策略 | 竞品分析✅ + 商业画布✅ |
| 留存分析 | 商业画布✅ + 北极星指标✅ |

## 六、练习题模块

### 6.1 核心设计

- **练习 = 方法论的刻意训练**，不是孤立的题目
- **双真实案例**：案例A=用户的想法/选的产品，案例B=用户选的真实竞品/参照
- **渐进提示强度**：根据累计得分动态调整「💡 给点提示」的输出强度
- **苏格拉底式引导**：用户卡壳时不给答案，用追问引导

### 6.2 提示强度档位（核心修正：按累计得分，不按练习次数）

> **设计原则**：不再数练了几次，只看累计拿了多少分。练得烂练100次也不该减少提示，练得好2次就该升级。

#### 6.2.1 档位定义

| 累计得分区间 | 档位 | 提示风格 | 给用户看的标签 | 给出什么 |
|------------|------|---------|-------------|---------|
| **0 - 3分** | 1档 | **强提示**（详细） | 「需要引导」 | 给出思考框架 + 分析角度 + 参考方向 + 具体案例 |
| **4 - 6分** | 2档 | **中等提示**（简要） | 「渐入佳境」 | 只给关键词 + 角度提醒 |
| **7 - 9分** | 3档 | **简单提示**（极简） | 「基本掌握」 | 只给一个方向性提醒 |
| **10分+** | 4档 | **无提示**（独立） | 「独立分析」 | 不提供任何提示 |

> 每次练习满分5分，累计得分永不清零，只增不减（除非触发遗忘降级）。

#### 6.2.2 完整示例

```
小明练「竞品分析」：

第1次：3.5分 → 累计3.5分 → 1档（强提示）
  系统提示：「分析目标用户时，从这几个角度入手：
    ①年龄层 ②使用场景 ③付费意愿 ④替代选择。
    先看看Forest在美国和印度的定价差多少？」

第2次：3.8分 → 累计7.3分 → 3档（简单提示）
  系统提示：「别忘了考虑用户分层维度」

第3次：4.2分 → 累计11.5分 → 4档（无提示）
  系统提示：无（完全独立）
```

#### 6.2.3 算法实现

```typescript
function getHintLevel(
  templateKey: string,
  skillProgress: SkillProgress
): number {
  const history = skillProgress.templateHistory[templateKey];
  const totalScore = history?.scores.reduce((a, b) => a + b, 0) ?? 0;
  const lastPracticed = history?.lastPracticed ?? 0;
  const daysSinceLastPractice = (Date.now() - lastPracticed) / (1000 * 60 * 60 * 24);

  // 1. 根据累计得分确定档位
  let level;
  if (totalScore <= 3)      level = 1;  // 强提示
  else if (totalScore <= 6) level = 2;  // 中等提示
  else if (totalScore <= 9) level = 3;  // 简单提示
  else                       level = 4;  // 无提示

  // 2. 遗忘降级（久不练，降一档防止遗忘）
  if (daysSinceLastPractice > 90)      level = Math.max(level - 3, 1);
  else if (daysSinceLastPractice > 60) level = Math.max(level - 2, 1);
  else if (daysSinceLastPractice > 30) level = Math.max(level - 1, 1);

  return level;
}

function getHintStyle(level: number): string {
  switch (level) {
    case 1: return '强提示';   // 详细引导 + 思考框架 + 参考方向
    case 2: return '中等提示'; // 关键词 + 角度提醒
    case 3: return '简单提示'; // 方向性提醒
    case 4: return '无提示';   // 完全独立
  }
}
```

#### 6.2.4 遗忘降级机制

| 间隔时间 | 降级幅度 | 说明 |
|---------|---------|------|
| ≤ 30天 | 不降 | 正常保持 |
| 30-60天 | 降1档 | 可能有些生疏 |
| 60-90天 | 降2档 | 需要较多复习 |
| > 90天 | 重置为1档 | 几乎忘记，从头引导 |

> **降级只影响提示强度，不影响累计得分。** 分还是那些分，只是系统认为你可能忘了，多给点提示。

### 6.3 中途思维引导（苏格拉底式）

**只在用户主动点击「💡 给点提示」时触发**，不主动弹出。

**每个维度独立计数**：竞品分析有6个维度，每个维度各自记录 `hintsUsed`。

**同一维度的追问递进**（每次点击对应的内容）：

| 点击次数 | 引导层级 | 示例 |
|---------|---------|------|
| 第1次 | 开放性问题 | 「先想想，Forest的用户打开App的场景有哪些？」 |
| 第2次 | 更具体的追问 | 「他们是在教室、宿舍、还是图书馆用的？」 |
| 第3次+ | 几乎给答案，但用问句 | 「是不是独处学习时最需要外部监督？」 |

**引导语气规则**：永远不给直接答案；基于用户已填写内容追问；暗含方法论结构；鼓励不打击。

function recommendNextTopic(skillProgress, recentBlindSpots) {

    !skillProgress.templateHistory[t.key]

## 附录：关键决策记录

| # | 决策 | 结论 |
|---|------|------|
| 1 | 方法论数量 | **7个**（新增约束权衡法） |
| 2 | 模板数量 | **29个**（24现有+5新增） |
| 3 | 模板-方法论映射 | **「1主+1辅」双绑定** |
| 4 | 独立训练案例 | **必须是真实产品** |
| 5 | 知识卡片定位 | **独立「每日方法论」入口** |
| 6 | 知识卡片推送频率 | **每日1张，3分钟阅读** |
| 7 | 盲点补齐 | 5个新增模板覆盖变现/留存/运营/技术可行性 |
| 8 | V2预留 | 5个高级模板 + 用户等级系统 |
| 9 | P0范围 | 仍只跑通competitive_analysis单模板 |
| 11 | 提示强度判定 | **按累计得分分4档**（0-3/4-6/7-9/10+），不按练习次数 |
| 12 | 遗忘降级 | 间隔>30天降1档，>60天降2档，>90天重置1档 |
| 10 | 两种模式关系 | 共享分数体系，随时可切换 |
