## 一、项目概述

### 1.1 产品定位

Web 思维训练工作台是一个基于 AI 的商业洞察力与产品思维刻意练习平台。核心理念是"练·学·看"的完整闭环：

- **练**：通过压力测试暴露思维弱点
- **学**：通过成长教练精准补强（知识卡片 + 练习题 + AI评分 + 模拟追问）
- **看**：通过对话考古复盘整个决策链条

### 1.2 目标用户

| 用户类型 | 描述 | V1支持 | V2预留 |
|---------|------|--------|--------|
| 在校学生 | 需要培养商业思维 | ✅ | |
| 转行上班族 | 想转到产品经理/运营的职场人士 | ✅ | |
| 初级产品从业者 | 需要培养商业分析能力 | ✅ | |
| 3-5年经验PM | 需要系统提升战略能力 | | ✅ |

### 1.3 核心价值主张

- **一句话宣言**：你的刻意练习圣地
- **差异化**：29个权威模板 + 7套可迁移方法论 + AI追问机制 + 练习即构建（自动生成项目文档）+ 成长仪表盘
- **明确反对**：反对功能堆砌、反对信息过载、反对虚拟案例练习
- **核心原则**：练习即构建，方法论即结构

### 1.4 三大模块概览

| 模块 | 核心价值 | 目标 | 关键特征 |
|------|---------|------|---------|
| **练：压力测试** | 暴露思维弱点 | 有想法需要被挑战 | 三轮致命追问 + 盲区报告 |
| **学：成长教练** | 精准补强 | 某个领域不懂需要学 | 每日方法论 + 知识卡片 + 练习题 + AI评分 + 模拟追问 + 练习即构建 |
| **看：对话考古** | 复盘决策链 | 想整理往期对话 | 五维分析 + 报告生成 + 一键归档 |

### 1.5 关键设计决策

| # | 决策 | 结论 |
|---|------|------|
| 1 | 方法论数量 | **7个**（新增约束权衡法） |
| 2 | 模板数量 | **29个**（24现有 + 5新增补齐盲点） |
| 3 | 模板-方法论映射 | **「1主+1辅」双方法论绑定** |
| 4 | 独立训练案例 | **必须是真实产品**（非虚拟场景） |
| 5 | 知识卡片定位 | **独立「每日方法论」入口**（非练习附属品） |
| 6 | 提示强度判定 | **按累计得分分4档**（0-3/4-6/7-9/10+），不按练习次数 |
| 7 | 遗忘降级机制 | 间隔>30天降1档，>60天降2档，>90天重置1档 |
| 8 | 练习表单 | **始终空白，不预填**（只在点击提示时给引导） |
| 9 | 盲点补齐 | 5个新增模板覆盖变现/留存/运营/技术可行性 |
| 10 | V2预留 | 5个高级模板 + 用户等级系统 |

## 二、压力测试引擎（练）

### 2.1 功能定义

模拟大厂面试官追问逻辑，对用户输入的任何商业 idea 进行多轮致命追问，暴露思维盲区。训练结果自动归档为结构化项目树，形成个人知识资产。

### 输入

- 用户用自然语言描述一个商业 idea（可以很简略，如"我想做一个帮助人专注的 App"）
- 用户可以选择追问深度：快速（6轮）/ 标准（12轮）/ 深度（20轮）

### 输出

- AI 三轮致命追问：每轮不同维度，逐层加深
- 思维盲区报告：关键漏洞清单，带修复建议
- 结构化归档：自动存入对应模板（竞品分析/市场规模/PRD 等）

### 2.2 用户交互流程

| 步骤 | 动作 | 说明 |
|------|------|------|
| 1 | 用户描述 idea | 输入框，支持自然语言 |
| 2 | 选择深度 | 快速/标准/深度三档切换 |
| 3 | 第一轮追问（定位） | 5 个关于用户/痛点/价值的追问 |
| 4 | 第二轮追问（竞争） | 5 个关于竞品/差异化/壁垒的追问 |
| 5 | 第三轮追问（可行性） | 5 个关于商业模式/风险/路线图的追问 |
| 6 | 盲区报告 | 可视化展示思维弱点分布 |
| 7 | 一键归档 | 存入项目树对应模块 |
| 8 | 触发成长教练 | 盲区报告自动推荐学习主题 |

### 2.3 AI Prompt 设计

**第一轮：定位追问**

- "你说目标用户是'重度用户'——每天解锁100次以上算重度，还是使用4小时以上算重度？"
- "你的核心痛点是'专注力被碎片化App毁掉'——这是你猜的，还是用户说的？"

**第二轮：竞争追问**

- "你说差异化是'建筑感空间'——这个差异化是比 Forest 好在哪里？"
- "如果谷歌明天复制你的功能，你的壁垒在哪里？"

**第三轮：可行性追问**

- "你的定价策略是什么？用户愿意为这个价值付多少？"
- "如果 6 个月后数据不达预期，你的 Plan B 是什么？"

### 2.4 盲区报告 → 触发成长教练

压力测试完成后，盲区报告自动传递给成长教练模块：

```
压力测试产出：盲区报告
  ├── "你的竞品分析有盲区——只分析了功能，没分析定价策略"
  │     → 成长教练推荐：「定价策略」模板
  │
  ├── "你对目标用户的描述太模糊"
  │     → 成长教练推荐：「用户画像」模板
  │
  └── "没有考虑技术可行性"
        → 成长教练推荐：「技术可行性」模板
```

### 2.5 开发建议

| 优先级 | 开发项 | 说明 |
|--------|--------|------|
| P0 | 三轮追问的 Prompt 设计与切换逻辑 | 核心体验，决定质量 |
| P0 | 追问结果的可视化展示（答案/新问题高亮） | 用户需要看到进度 |
| P0 | 盲区报告 → 成长教练的触发链路 | 打通"练→学"闭环 |
| P1 | 追问深度三档切换 | 提升体验曲线 |
| P1 | 盲区报告生成 | 让用户看见自己的弱点 |
| P2 | 历史追问对比（第二次比第一次好在哪里） | 成长感 |

## 三、成长教练（学）

### 3.1 模块架构

成长教练包含六个子模块：

```
成长教练
  │
  ├── 📚 每日方法论（独立内容入口）
  │     ├── 今日推送（1张卡片，3分钟阅读）
  │     ├── 方法论归档（7个方法论 × N个案例）
  │     └── 我的收藏
  │
  ├── 📖 知识卡片（练习前预习）
  │     ├── Level 1：一句话概念
  │     ├── Level 2：一个具体案例
  │     ├── Level 3：方法论总结（差异化呈现）
  │     └── Level 4：自检清单
  │
  ├── ✏️ 练习题（核心环节）
  │     ├── 双真实案例（用户想法 + 真实竞品）
  │     ├── 渐进提示强度（按累计得分分4档）
  │     └── 苏格拉底式引导（用户主动触发）
  │
  ├── 📊 AI评分反馈
  │     ├── 逐维度1-5分
  │     ├── 最大盲区指出
  │     ├── 方法论反思（关联历史表现）
  │     └── 下一步建议
  │
  ├── 🎤 模拟面试追问
  │     ├── 针对评分最低维度
  │     ├── 答不上来→引导，答对→追问更深
  │     └── 复用LiveLab（expertMode='guided'）
  │
  └── 📈 能力雷达图
        ├── 7维度方法论掌握度
        ├── 历史对比
        └── 智能推荐下一主题
```

### 3.2 两种训练模式

成长教练支持两种训练模式，**必须并存**：

| 维度 | 项目绑定训练 | 独立训练 |
|------|------------|---------|
| **场景** | 我有一个想法，想系统性想清楚 | 我想单独提升某个能力 |
| **案例A（主体）** | 用户自己的项目/想法 | 用户选的真实产品（自己常用或行业标杆） |
| **案例B（参照）** | 用户选的真实竞品 | 用户选的分析对象 |
| **场景真实性** | ✅ 双真实 | ✅ 双真实 |
| **产出** | 完整项目文档 → 保存到项目树 | 完整分析报告 → 保存到练习记录 |
| **解锁规则** | 有依赖关系（前置模板解锁） | 无依赖，29模板全部可选 |
| **进阶训练** | ✅ 4档提示强度 | ✅ 4档提示强度 |
| **雷达图更新** | ✅ +1 | ✅ +1 |
| **「绑定到项目」按钮** | 不需要 | ✅ 随时可点击 |

**两种模式共享**：方法论维度分数、4档提示强度体系、AI评分与反馈逻辑。

### 3.3 七大方法论体系

#### 3.3.1 方法论总览

| # | 方法论 | 一句话 | 记忆钩子 | 呈现方式 | 名称位置 |
|---|--------|--------|---------|---------|---------|
| 1 | **多维拆解法** | 把复杂对象拆成维度逐个分析 | 同时想多个方面 | 场景类比 | 开头 |
| 2 | **场景代入法** | 进入用户场景感受friction找痛点 | 进入那个场景 | 追问引导 | 中间 |
| 3 | **第一性推理** | 追问为什么追到根本原因 | 追到根上 | 对比反差 | 结尾 |
| 4 | **推演博弈法** | 假设对手反应推演多步选最优 | 填完3轮就知道 | 工具化（决策表） | 给工具时 |
| 5 | **闭环验证法** | 假设→最小验证→数据→迭代 | 建一点测一点 | 故事化 | 高潮处 |
| 6 | **指标拆解法** | 拆核心指标找杠杆 | 拆开找杠杆 | 工具化（拆解树） | 解决后 |
| 7 | **约束权衡法**（新增） | 资源有限先做哪件 | 先做ROI最高的 | 工具化（ICE矩阵） | 给工具时 |

#### 3.3.2 方法论呈现方式（避免千篇一律）

**多维拆解法 — 场景类比（开头给名字）**：

> 先说一个你本来就会的本能：挑手机时，你会同时想「贵不贵？好不好买？拍照好不好？用久了卡不卡？」这个「同时想多个方面」的本能就是【多维拆解】。$APPEALS只是把它变成了8个维度。

**第一性推理 — 对比反差（结尾点破）**：

> 先看一个反例：「这个App需要加社交功能」（这是结论还是推理？）再看一个正例：「用户说孤独 → 孤独=需要连接？→ 连接=社交功能？→ 连接=被看见？→ 也许『展示』比『聊天』更重要」。区别在哪？前者停在表面，后者不停追问「为什么」。这种「追到根上」的思考方式，我们叫它【第一性推理】。

**约束权衡法 — 工具化（ICE矩阵）**：

> 你只有100万预算、3个月时间、5个人。想法有10个，先做哪个？填这个ICE表：Impact × Confidence × Ease。总分高的先做。这个工具叫【约束权衡法】。

### 3.5 每日方法论（独立内容入口）

#### 3.5.1 定位

知识卡片**不是练习的附属品**，而是独立的每日内容产品。即使用户不练习，也能每天获得思维训练。

#### 3.5.2 两种形态

| 形态 | 场景 | 内容 | 时长 |
|------|------|------|------|
| **每日推送** | 用户打开成长教练 | 今日行业热点 + 方法论拆解 | 3分钟 |
| **知识卡片** | 练习前预习 / 主动查阅 | 完整四层结构 | 5-10分钟 |

#### 3.5.3 每日推送示例

```
📅 2026-05-15 每日方法论 · 第47期

🔥 今日行业热点：小红书切断外链，力推站内闭环电商

💡 用「第一性推理」拆解：
   为什么小红书不做导流，要做闭环？
   → 追问1：导流的本质是什么？（流量变现，一次性）
   → 追问2：闭环的本质是什么？（用户资产沉淀，持续性）
   → 追问3：小红书的终局是什么？（内容+交易的超级生态）

💭 今日自检：如果你是拼多多PM，你会怎么应对？

[深入了解第一性推理 →]  [用这个方法练一次 →]
```

#### 3.5.4 知识卡片四层结构

| 层次 | 内容 | 说明 |
|------|------|------|
| **Level 1** | 一句话概念 | 用大白话解释，不要术语 |
| **Level 2** | 一个具体案例 | 锚定应用场景，真实商业案例 |
| **Level 3** | 方法论总结 | 差异化呈现（因方法论而异） |
| **Level 4** | 自检清单 | 3-5个检查点 |

#### 3.5.5 内容生成策略

- **热门知识点**：预配置（`knowledgeTopics.js`），加载零延迟
- **每日推送**：AI实时生成（结合当日行业热点）
- **长尾知识点**：AI实时生成，缓存结果

### 3.6 练习题模块

#### 3.6.1 核心设计

- **练习 = 方法论的刻意训练**，不是孤立的题目
- **双真实案例**：案例A=用户的想法/选的产品，案例B=用户选的真实竞品
- **练习表单始终空白**，只在点击「💡 给点提示」时按档位给出提示
- **苏格拉底式引导**：用户卡壳时不给答案，用追问引导

#### 3.6.2 提示强度档位（按累计得分，不按练习次数）

> **核心原则**：不再数练了几次，只看累计拿了多少分。练得烂练100次也不该减少提示，练得好2次就该升级。

| 累计得分区间 | 档位 | 标签 | 提示风格 | 给出什么 |
|------------|------|------|---------|---------|
| **0 - 3分** | 1档 | 需要引导 | **强提示**（详细） | 思考框架 + 分析角度 + 参考方向 + 具体案例 |
| **4 - 6分** | 2档 | 渐入佳境 | **中等提示**（简要） | 关键词 + 角度提醒 |
| **7 - 9分** | 3档 | 基本掌握 | **简单提示**（极简） | 方向性提醒 |
| **10分+** | 4档 | 独立分析 | **无提示**（独立） | 不提供任何提示 |

每次练习满分5分，累计得分永不清零，只增不减（除非触发遗忘降级）。

**完整示例**：

```
小明练「竞品分析」：

第1次：3.5分 → 累计3.5分 → 1档（强提示）
  系统提示：「分析目标用户时，从这几个角度入手：
    ①年龄层 ②使用场景 ③付费意愿 ④替代选择。
    先看看Forest在美国和印度的定价差多少？」

第2次：3.8分 → 累计7.3分 → 3档（简单提示）
  系统提示：「别忘了考虑用户分层维度」

第3次：4.2分 → 累计11.5分 → 4档（无提示）
  系统提示：无（完全独立）
```

#### 3.6.3 遗忘降级机制

| 间隔时间 | 降级幅度 | 说明 |
|---------|---------|------|
| ≤ 30天 | 不降 | 正常保持 |
| 30-60天 | 降1档 | 可能有些生疏 |
| 60-90天 | 降2档 | 需要较多复习 |
| > 90天 | 重置为1档 | 几乎忘记，从头引导 |

降级只影响提示强度，**不影响累计得分**。

#### 3.6.4 算法实现

```typescript
function getHintLevel(
  templateKey: string,
  skillProgress: SkillProgress
): number {
  const history = skillProgress.templateHistory[templateKey];
  const totalScore = history?.totalScore ?? 0;
  const lastPracticed = history?.lastPracticed ?? 0;
  const daysSince = (Date.now() - lastPracticed) / (1000 * 60 * 60 * 24);

  // 1. 根据累计得分确定档位
  let level;
  if (totalScore <= 3)      level = 1;
  else if (totalScore <= 6) level = 2;
  else if (totalScore <= 9) level = 3;
  else                       level = 4;

  // 2. 遗忘降级
  if (daysSince > 90)      level = Math.max(level - 3, 1);
  else if (daysSince > 60) level = Math.max(level - 2, 1);
  else if (daysSince > 30) level = Math.max(level - 1, 1);

  return level;
}
```

#### 3.6.5 苏格拉底式引导

**只在用户主动点击「💡 给点提示」时触发**，不主动弹出。

**每个维度独立计数**，同一维度的追问递进：

| 点击次数 | 引导层级 | 示例 |
|---------|---------|------|
| 第1次 | 开放性问题 | 「先想想，Forest的用户打开App的场景有哪些？」 |
| 第2次 | 更具体的追问 | 「他们是在教室、宿舍、还是图书馆用的？」 |
| 第3次+ | 几乎给答案，但用问句 | 「是不是独处学习时最需要外部监督？」 |

**引导语气规则**：永远不给直接答案；基于用户已填写内容追问；暗含方法论结构；鼓励不打击。

#### 3.6.6 小白用户的三种"项目"来源

| 来源类型 | 示例 | 练习如何展开 |
|---------|------|-------------|
| **一个模糊的想法** | 「我想做一个帮助人专注的App」 | 系统帮他细化，练习就是细化的过程 |
| **一个天天用的产品** | 天天用美团外卖 | 「假设你是美团PM，要改进一个痛点」 |
| **一个观察到的现象** | 身边很多老人不会用健康码 | 「如果让你做一个老人友好的健康码产品」 |

### 3.7 AI评分反馈模块

#### 3.7.1 评分标准

| 分数 | 标准 | 说明 |
|------|------|------|
| 5分 | 深度洞察，有独特视角 | 超出预期 |
| 4分 | 分析完整，略有遗漏 | 掌握良好 |
| 3分 | 基础正确，缺乏深度 | 理解概念 |
| 2分 | 方向正确，理解片面 | 有思路但漏维度 |
| 1分 | 误解概念 | 需重新学习 |

#### 3.7.2 反馈结构（五部分）

1. **总体评分**：数字 + 渐变色进度条
2. **逐维度点评**：5个小圆点表示
3. **最大盲区**：最需要改进的点
4. **方法论反思**：主方法论+辅方法论双反思
5. **下一步建议**：模拟追问 / 重练 / 下一主题

#### 3.7.3 方法论反思示例

```

  你分析了Forest的定价和功能，但没有预判：
```

### 3.8 模拟面试追问模块

**位置**：每次练习评分后，用户点击「接受检验」进入。

## 四、对话考古（看）

### 4.1 功能定义

对历史对话进行多层级挖掘——从时间轴整理、决策点提取，到认知盲区发现、决策复盘报告自动生成。将碎片化讨论转化为可归档的结构化资产。

### 输入

- 用户创建考古会话（命名，如"Thinking Lab 决策复盘"）
- 多次追加粘贴对话记录，AI 合并分析

### 输出

- 五维分析报告：时间轴/转折点/盲区/假设/知识资产
- 每个条目有✅确认/❌驳回/✏️修改 三个操作
- AI 智能推荐归档目标，用户一键确认
- 最终生成 Markdown 格式的复盘报告

### 4.2 用户交互流程

| 步骤 | 动作 | 说明 |
|------|------|------|
| 1 | 创建考古会话 | 命名 + 初始化空状态 |

| 3 | AI 分析 | 五维分析，条目显示为待确认 |
| 4 | 人工审阅 | 逐条确认/驳回/修改 |
| 5 | 追加新对话（可选） | AI 合并分析，保留已确认 |
| 6 | 生成最终报告 | Markdown 格式，可复制到飞书/Notion |
| 7 | 一键归档 | AI 推荐模板，用户确认 |

### 4.3 五维分析架构

| 维度 | 输出 | 使用场景 |
|------|------|---------|
| 时间轴 | 关键时间节点 | 看想法怎么一步步演进的 |
