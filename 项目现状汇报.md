## 一、项目总览

| 维度 | 详情 |
|------|------|
| **项目名** | web-lab（Thinking Lab / Kairos 思维实验室） |
| **技术栈** | React 18 + Vite 5 + TailwindCSS 3 + Framer Motion |
| **路由方案** | 无路由库，通过 `labMode`（live/archaeology）状态切换视图 |

### 4.4 ⚠️ 两套数据层并存问题

项目中存在 **两套并存的数据层**，这是当前最大的架构风险：

| 维度 | 数据层 A：LabContext + useReducer | 数据层 B：dataStore.js（store / archaeologyStore） |
|------|----------------------------------|-----------------------------------------------------|
| **存储 key** | `kairos-project-tree`（通过 useLocalStorage 同步） | `kairos-lab-data`（store）+ `kairos-archaeology-sessions`（archaeologyStore） |
| **操作方式** | 通过 Context 方法（`createDocument`、`saveDocument` 等） | 直接调用 `store.createDocument()`、`archaeologyStore.xxx()` |
| **使用者** | Sidebar、LabPanel、ArchivePanel、DocumentDetail 等 | ArchaeologyReportView、ArchaeologySessionList |
| **同步机制** | ❌ **无自动同步** | ❌ **无自动同步** |

**具体冲突点：**
1. `ArchaeologyReportView` 归档资产时调用 `store.createProject()` / `store.createDocument()`，写入 `kairos-lab-data`
2. 但 Context 的项目树存储在 `kairos-project-tree`，两者 **互不可见**
3. `initStore()` 在 App.jsx 中调用，但仅用于初始化，后续操作走 Context
4. `LabContext` 的 `archiveToProject` 方法正确地走 Context，但 `ArchaeologyReportView` 没有使用它

## 七、下一步建议

基于当前现状，推荐按以下优先级推进：

### 优先级 1：修复数据层断裂（R1）

**目标**：统一数据访问路径，消除归档数据丢失问题。

- 将 `ArchaeologyReportView` 中的 `store.createProject()` / `store.createDocument()` 替换为 Context 的 `archiveToProject()` / `addDocumentToCategory()`
- 评估是否完全弃用 `dataStore.js` 中的 `store` 对象（仅保留 `archaeologyStore` 和 `settingsStore`）
- 或者反过来：让 Context 的项目树读写也走 `store`，消除双写

### 优先级 2：修复 expertMode bug + 模式切换一致性（R2 + R4）

**目标**：确保压力测试和成长教练模式能正确切换。

- 在 `LabPanel.LiveLab` 中确认 `expertMode` 的来源（从 `useLab()` 解构或从 props 传递）
- 统一 `labMode` / `activeLabTab` 的命名，建议全部使用 `labMode`（`'live'` / `'archaeology'`）
- 修复 `ArchivePanel` 中 `setActiveLabTab('practice')` → `setLabMode('live')`

### 优先级 3：清理 V1 考古残留 + 提示词去重（R3 + R5）

**目标**：减少代码冗余，降低维护成本。

- 移除 `ArchaeologyTimeline.jsx`（V1）或明确标记为 deprecated
- 移除 `ArchaeologySessionList.jsx`（功能已被 `ArchaeologyV2` 内联列表取代）
- 将 `ArchaeologyInputArea` 中的内联提示词替换为 `import { ARCHAEOLOGY_V2_PROMPT } from '../config/aiPrompts'`
- 清理所有 `console.log` 调试日志和 CommonJS 兼容导出
